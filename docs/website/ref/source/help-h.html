Help

Class : system operation, defined in defs.ndf
Usage : help Topic
See : symbols

The operation help reads a text file from directory nialhelp and displays the information in an editwindow. To obtain help on "tally;" for example, type:

help "tally;

After reading the help message, you can press <Enter;> and enter another topic. In this case, do not put the phrase mark before the word; just type the word. Alternately, you can press <Esc;> and exit the help operation.

All the topics can be displayed and one chosen by:

help "topic;

If you press <Enter;> after reading help on a topic from the list of all topics, the list of topics is displayed again with the one just read at the top of the list. You can press <Esc;> to exit the help or use <Page; Up>, <Page; Down> or the Arrow keys to move the cursor to another topic. The list of topics contains some helpful hints on how to handle typical requirements using Q'Nial and explains some Nial topics by quoting from the manuals.

If you are using "help;" in a program and wish to limit the display to help on the one topic, include a second argument of 1 or l. Then, when <Enter;> or <Esc;> are pressed, the help operation ends.

Hitch

Class : binary construction operation
Usage : A hitch B hitch A B
See : append, link, list, solitary

The operation hitch attaches A to the front of the list of items of B. It returns a list of length one greater than the tally of B.

     (2 3 4) hitch (5 6 7)
+-----+-+-+-+
|2 3 4|5|6|7|
+-----+-+-+-+
     7 hitch 3
7 3
     hitch 'Wow' ''
+---+
|Wow|
+---+

The first example shows that the list 2 3 4 becomes an item on the front of the list 5 6 7. In the next example, the right argument of hitch is treated as a list. The last example shows that if the right argument is an empty list, the result is the left argument as a solitary.

Definition 
hitch IS OPERATION A B {
   solitary A link B }

Equations 
A hitch B = A hitch (list B)

A hitch Null = solitary A

list (A hitch B) = A hitch B

not empty A ==> first A hitch rest A = list A

shape A reshape (first A hitch rest A) =f A

Host

Class : system operation
Usage : host S
See : edit, refresh

The operation host executes S as a host command language instruction.

The argument S is a string or a phrase. If the action carried out by the host system produces output, the output is displayed on the screen. In window mode, Q'Nial is unable to capture this output and hence it may scramble the screen output. The screen is restored by executing Refresh.

The result of the host command is the fault ?noexpr if the command has returned normally; or a fault generated from a system dependent error message supplied by the host operating system.

At the top level loop, a line beginning with !, the exclamation mark, is interpreted as a host command.
