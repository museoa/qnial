Blend

Class : binary axis rearrangement operation, a primitive
Usage : A blend B blend A B
See : split, mix, partition, rows, cols, raise

The operation blend combines the top two levels of an array B into a single level blending the axes of the second level of B into the combined level according to the axis numbers given in A. The items of B must be of the same shape.

If the array B is equivalent to A split C for some array C, the result of A blend B is C.

The tally of A is the valence of an item of B. The items of A indicate where the axes of the items of B are placed in the shape of the result.

     C := (2 4 3 reshape count 24) ;
     B := 2 0 split C
+----+----+----+-----+
|1 13|4 16|7 19|10 22|
|2 14|5 17|8 20|11 23|
|3 15|6 18|9 21|12 24|
+----+----+----+-----+

     C := 2 0 blend B
 1  2  3   13 14 15
 4  5  6   16 17 18
 7  8  9   19 20 21
10 11 12   22 23 24

In the example, B is a list of four tables of shape 3 2 created by a 2 0 split. The result of 2 0 blend B is the array of valence 3 that is the second argument to split.

Equations 
SORT <= list I = axes A and not empty A ==> I blend (I split A) = A

equal EACH shape A and not empty A ==> (valence A+axes first A)blend A=mix A

A blend B =f [Null,A] PARTITION first B

Break

Class : system expression
Usage : Break
See : resume, step, callstack, set "trace, setdeftrace

The execution of break causes the interpreter to interrupt normal execution inan expression sequence and to prompt for input with the prompt "--;>".; The visible environment is that of the expression in which the break occurs. Thus, it is possible to examine the values of local variables in break mode.

The expression resume is used to return from break mode. The expression step displays the next expression to be evaluated in the expression sequence where break occurs, evaluates it with trace on and then returns to break mode at the next item in the expression sequence. If step is entered at the end of an expression sequence, the computation resumes.

Break mode can also be entered by typing <Ctrl; B> in response to the prompt for a read or readscreen operation or readchar expression. In this case, after resume or step is entered, Q'Nial returns to the read or readscreen operation and awaits the user input. Break mode cannot be entered by typing <Ctrl; B> in editwindow or when setinterrupts has been used to inhibit interrupts.

     Break
-->break; debug mode, type RESUME to exit
-->Resume;
-->returning; from break debug mode

Bycols

Class : application transformer
Usage : BYCOLS f A
See : Bycols, partition

The transformer bycols applies an operation f to the columns of an array A, where f is an operation that maps lists to lists.

     setformat '%5.3f';
     BYCOLS (SORT <=)
        (5 6 reshape random 30)
0.726 0.009 0.210 0.385 0.123 0.119
0.851 0.448 0.384 0.476 0.193 0.384
0.880 0.449 0.536 0.603 0.455 0.498
0.882 0.729 0.631 0.819 0.639 0.521
0.911 0.893 0.810 0.961 0.940 0.672

The example generates a random table and then sorts each column.

Definition 
BYCOLS IS TRANSFORMER f OPERATION A {
   [valence A - 2 max 0] PARTITION f A }

Equation 
shape f A = shape A ==> shape BYCOLS f A = shape A

Bye

Class : system expression
Usage : Bye
See : Continue, Save, Restart

The expression bye is used to terminate a session of Q'Nial and to return to the host environment. The current workspace is not saved.

Byrows

Class : application transformer
Usage : BYROWS f A
See : Bycols, Partition

The transformer byrows applies an operation f to the rows of an array A, where f is an operation that maps lists to lists.

     BYROWS reverse (5 6 reshape count 30)
 6  5  4  3  2  1
12 11 10  9  8  7
18 17 16 15 14 13
24 23 22 21 20 19
30 29 28 27 26 25

The example reverses the rows of the generated table.

Definition 
BYROWS IS TRANSFORMER f OPERATION A {
   [valence A - 1 max 0] PARTITION f A }

Equation 
shape f A = shape A ==> shape BYROWS f A = shape A
