Unequal

Class : predicate operation
Usage :
                     A unequal B      A ~= B
                     unequal A
See : equal, diverse, notin

The operation unequal is normally used to determine that two arrays A and B are not identical. Two nonempty arrays are identical if they have identical shapes and hold identical items at each location. Two empty arrays are identical if they have identical shapes. The operation returns true if A is not identical to B and false otherwise. It is extended to arrays with more than two items in the same manner as the operation equal is, being false only if all the items are equal.

     2 3 4 ~= [2,3,4]
o

     Null ~= ''
o

     2 (3 4) ~= (2 3) 4
l

     unequal (2 3) (2 3) (2 3)
o
Definition 
unequal IS OPERATION A { not equal A }

Update

Class : evaluation operation
Usage : update Nm I A
See : updateall, deepupdate, place

The operation update provides the semantics of the Nm@I := A form of assignment expression. Nm must be an existing variable represented by a string, phrase or a cast. I is the address of the location to be updated. A is the array to be placed in the variable.

     X := 3 4 5; update "X; 0 8
8 4 5
     update !X 2 "goodbye;
8 4 goodbye

The major purpose of update is to allow a selective update at one location in the array associated with a global variable without forcing a copy. By passing the name of the variable to the operation that is doing the update, rather than its value, no sharing of the internal data is made and hence the update can be "in; place".;

Updateall

Class : evaluation operation
Usage : updateall Nm I A
See : update, deepupdate, placeall

The operation updateall provides the semantics of the Nm#I := A form of assignment expression. Nm must be an existing variable represented by a string, phrase or a cast. I is the array of addresses of the locations to be updated. A is the array of values to be placed in the variable.

     X := 3 4 5;
     updateall "X; (0 2) (8 10)
8 4 10
     updateall !X (1 2) ("hello; "goodbye;)
8 hello goodbye

The major purpose of updateall is to allow a selective update at several locations in the array associated with a global variable without forcing a copy. By passing the name of the variable to the operation that is doing the update, rather than its value, no sharing of the internal data is made, and hence the update can be "in; place".;
