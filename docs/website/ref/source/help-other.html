Unary pervasive

Class : unary pervasive arithmetic operation, a primitive
Usage : f A
See : binary, multi

Each operation f in this class maps an atom to an atom.

A unary pervasive operation maps an array to another array with identical structure, mapping each atom by the function's behaviour on atoms. All of the scientific operations and the unary operations of arithmetic and logic are unary pervasive.

The scientific operations are implemented using the library routines provided with the C compiler used to construct Q'Nial. The accuracy of the result is determined by the precision of the floating point number system of the computer and the accuracy of the library routine approximation.

Binary Pervasive

Class: operation class
Usage: A f B f A B
See : unary pervasive, multi pervasive

Each operation f in this class maps a pair of atoms to an atom. The behaviour of each operation on pairs of atoms is described under its entry in section 3.3 . Here we describe the general properties.

A binary pervasive operation maps two arrays having identical structure to one with the same structure, mapping each pair of corresponding atom by the function's behaviour on pairs of atoms.

All of the binary operations of arithmetic and logic are binary pervasive.

If a binary pervasive operation is applied to a pair of arrays that do not have the same shape, then the effect is to build a conformable pair by replicating an atom or solitary item of the pair to the shape of the other item. If both items have tally different from one and are of unequal shape then the fault ?conform is returned. The replication of an item of tally one provides binary pervasive operations with a scalar extension capability. For example, 3 4 5 6 - 5 is equivalent to 3 4 5 6 - 5 5 5 5 .

If a binary pervasive operation is applied to an array that is not a pair, then a fault is returned.

Multi Pervasive

Class: operation class
Usage: f A
See : unary pervasive, binary pervasive
Some of the operations of Nial that operate on simple arrays are extended to arbitrarily nested arrays by being applied to the atoms at the deepest level. These are called pervasive operations. There are three classes of pervasive operations: unary pervasive, binary pervasive and multi pervasive. When a multi pervasive operation, such as sum, is applied to an array A with items all of identical structure, it returns an array of that same structure. The atoms of the result are computed by applying sum to the simple arrays formed by taking the atoms from positions in the corresponding items of the argument. For example, if T is a table, sum rows T returns the column totals for T. T 1 2 3 4 5 6 7 8 rows T +-------+-------+ |1 2 3 4|5 6 7 8| +-------+-------+ sum rows T 6 8 10 12 A multi pervasive operation used on a pair behaves in precisely the same manner as a binary pervasive operation. A multi pervasive operation f satisfies the equations f A = EACHALL f A f A = REDUCE f A In binary pervasive and multi pervasive operations, the items of the argument must all be of the same shape and structure. If they are not, items that have only one item, eg. atoms, are reshaped to the common shape. If all items with more than one item are not the same shape, the fault ?conform is returned. On a pair, a multi pervasive operation behaves the same way as a binary pervasive operation does. Modification of the argument using these binary and multi pervasive operations is the same as that done by the operation pack, except that it is done implicitly. Each operation in this class reduces a simple array (an array of atoms) to an atom. A multi pervasive operation maps an array with items having identical structure to one with the same structure, mapping each array of atoms in corresponding positions in the items by the function's behaviour on simple arrays. There are six operations in this class. They are the reductive operations of arithmetic and logic.
